6 条比较清晰的业务链条：

**1. 数据汇聚治理链**
多源数据整合与治理 -> 总览驾驶舱 -> 各专题分析模块

这条链是底座。先把站点台账、充电记录、区域负荷、街道乡镇、用户行为等数据统一接入和治理，再把标准化结果供给后面的画像、预测、调节和决策模块。

**2. 负荷认知与预测链**
多维度负荷特征画像构建 -> 充电负荷预测 -> 总览驾驶舱 / 辅助决策

先识别负荷的时段特征、波动特征、聚类结果，再做超短期和中长期预测，最后把预测结果用于预警、趋势展示和调度判断。

**3. 用户运营分析链**
重卡用户群体画像与行为预测 -> 各站点短期需求预测 -> 辅助决策数据应用

这条链关注“谁在充、什么时候充、去哪里充”。先做用户分群，再看各充电站的群体分布和短期需求，最后输出站点引导、分流和运营策略。

**4. 柔性调节执行链**
重卡用户群体画像与行为预测 -> 调节潜力分析与量化评估 -> 辅助决策数据应用

核心是把“用户可调行为”转成“可执行调节能力”。利用重卡停车时间长、夜间停车的特点，评估哪些订单可平移、哪些站点可柔调，再生成移峰填谷、夜谷补能、限制高峰到站等策略。

**5. 空间选址建设链**
地图充电站管理 -> 地图选址分析 -> 辅助决策数据应用

先看现有站点分布、负荷状态和利用率，再下沉到街道/乡镇看覆盖情况、增长趋势和薄弱区域，最后形成新增布点、扩容增桩、优化覆盖的决策依据。

**6. 预警处置闭环链**
充电负荷预测 + 乡镇细化研判 + 站点运行状态 -> 预警通知 -> 站点策略 -> **空间视图跟踪**
